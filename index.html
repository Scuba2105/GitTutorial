Git Tutorial

Push a New Project to GitHub

Introduction

This tutorial is aimed at improving knowledge of git and GitHub whilst also gaining more expeience at web development. By the end of this tutorial you will have learned how to create a new GitHub remote repository and how to push an existing project to this remote repo

Step 1- Create a new GitHub Repository

Sign in to GitHub and click the plus sign dropdown in the top right corner. Select the option to create a new repository. See below screenshot for the form that must be completed to create the new repository

IMPORTANT! You must make note of your GitHub username and the name given to the repository as it is required in later steps.

Step 2- Initialise Git in the Project Folder

You must first be in the root directory for the project you wish to push to GitHub. This step creates a hidden .git directory in the project folder which the git software recognises and uses to store all meta data and version history. Once is the root directory enter the following command at the terminal:

Note: If you have already initialised a Git repository for this project you can skip this step.

$ git init

Step 3- Add the Files to Git Staging Area

The git add command adds the specified files to the git staging area ready to be committed. There are different available arguments to use for this step outlined below.

$ git add -A

The -A argument tells git to stage all files for the next commit.

$ git add .

The . argument tells git to stage all changes in the root directory for the next commit.

$ git add "file"

The "file" argument tells git to stage the named file/s for the next commit.
